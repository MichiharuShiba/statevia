## 設計メモ

- `$.context` は実行時にのみ確定する値を扱うため、コンパイル時検証と実行時解決の分離が必要。
- `$.env` は secrets 取り扱いの設計が必要（参照可能キーの制限、監査ログの伏せ字、テナント境界）。
- v1.0.0 では未対応。ユーザー定義状態コード側で `StateContext` / 環境変数 API を使う回避策を維持する。
