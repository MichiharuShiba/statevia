# Wait 状態の DB 永続化（案メモ）

- Version: 1.0.0
- 更新日: 2026-04-10
- 対象: Engine / Core-API の Wait 実行モデル（将来検討）

---

## 現状の認識

- **Wait** は現状、**メモリ上**で保持されている。

## 方向性（案）

1. **Wait 状態を DB に保存**する。
2. メモリからは **Wait 中のワークフローインスタンスを削除**する（メモリに載せ続けない）。
3. **再開時**に DB から Wait 関連の状態を読み込み、**同一 `workflowId`** で Engine 上で実行を継続する。
