# Requirements: UI言語モード切替（日本語 / 英語）

## Introduction

本仕様は、UI 上で日本語と英語を切り替える機能を追加し、運用担当者が利用シーンに応じて表示言語を選択できるようにする。
対象は `services/ui/app` 配下の主要画面（共通ヘッダー、ハブ画面、詳細画面、実行画面）とする。

## Alignment with Product Vision

- 定義駆動運用の導線（Dashboard / Definitions / Workflows）を維持したまま、表示言語だけを切り替えられるようにする。
- 既存 API 契約や内部識別子（ID、ステータス値）を変更せず、UI 表示層で多言語化を完結させる。
- 現行の日本語 UI をデフォルトとして維持しつつ、英語表示を追加する。

## Requirements

### Requirement 1 — 言語切替操作の提供

**ユーザーストーリー:** 利用者として、画面上で言語を切り替えたい。なぜなら、利用者や利用環境に応じて表示言語を選びたいから。

#### Acceptance Criteria — Requirement 1

| No | アクター | きっかけ（ユースケース） | 期待される結果 |
| --- | --- | --- | --- |
| 1 | 利用者 | 共通ヘッダーを表示する | 日本語 / 英語の切替 UI が表示される |
| 2 | 利用者 | 日本語を選択する | 画面文言が日本語で表示される |
| 3 | 利用者 | 英語を選択する | 画面文言が英語で表示される |
| 4 | 利用者 | 同一 URL 上で言語を切り替える | 画面遷移せずに表示言語だけが更新される |

### Requirement 2 — 言語設定の保持

**ユーザーストーリー:** 利用者として、選択した言語を保持してほしい。なぜなら、画面遷移や再読み込みのたびに再設定したくないから。

#### Acceptance Criteria — Requirement 2

| No | アクター | きっかけ（ユースケース） | 期待される結果 |
| --- | --- | --- | --- |
| 1 | 利用者 | 言語を切り替える | 選択言語が永続化される（Cookie など） |
| 2 | 利用者 | ページを再読み込みする | 直前に選択した言語で表示される |
| 3 | 利用者 | 別画面へ遷移する | 遷移先でも同じ言語設定が適用される |
| 4 | システム | 未設定・不正な言語値を受け取る | 既定値（日本語）で安全に表示される |

### Requirement 3 — 文言辞書の分離と参照統一

**ユーザーストーリー:** 開発者として、文言を安全に追加・変更したい。なぜなら、言語追加時に影響範囲を追いやすくしたいから。

#### Acceptance Criteria — Requirement 3

| No | アクター | きっかけ（ユースケース） | 期待される結果 |
| --- | --- | --- | --- |
| 1 | 開発者 | 文言辞書を参照する | `ja` / `en` の辞書が同一型で管理される |
| 2 | 開発者 | 新しい文言キーを追加する | 型チェックで両言語の不足を検知できる |
| 3 | 開発者 | 画面で文言を利用する | 単一の取得口（Hook / Provider）から取得できる |
| 4 | 利用者 | UI を閲覧する | 画面内で言語混在が発生しない |

### Requirement 4 — ロケール依存表示の整合

**ユーザーストーリー:** 利用者として、言語だけでなく日時表記も自然に読める形で表示されてほしい。なぜなら、言語切替後に日時だけ日本語形式だと読みづらいから。

#### Acceptance Criteria — Requirement 4

| No | アクター | きっかけ（ユースケース） | 期待される結果 |
| --- | --- | --- | --- |
| 1 | 利用者 | 日本語表示で日時を閲覧する | `ja-JP` ロケール形式で表示される |
| 2 | 利用者 | 英語表示で日時を閲覧する | `en-US` など英語ロケール形式で表示される |
| 3 | 開発者 | 日時表示ロジックを追加する | ロケール取得を共通関数で再利用できる |

## Non-Functional Requirements

### Clarity

- 言語辞書、言語選択ロジック、画面適用箇所の責務を分離し、変更意図を追跡しやすくする。
- `requirements.md` / `design.md` / `tasks.md` で実装範囲と進捗を明確化する。

### Reliability

- 未設定値・不正値でも UI が破綻せず、既定言語で描画されること。
- 既存導線（Dashboard / Definitions / Workflows / Detail / Run / Graph）を維持すること。

### Performance

- 言語切替による不要な再フェッチ・過剰再描画を増やさないこと。
- 初期表示時の言語決定は軽量に行い、画面体感速度を悪化させないこと。

### Usability

- 切替 UI は常時アクセスしやすい位置（共通ヘッダー）に配置する。
- 現在選択中の言語が視覚的に判別できること。

## Out of Scope

- 日本語 / 英語以外の言語追加。
- API レスポンス自体の多言語化。
- 認証ユーザープロファイルへの言語設定保存。
