# Implementation Log: ui-playground P3.2

- **日付**: 2026-04-12
- **タスク**: `tasks.md` — P3.2 SSE トグルと UX 仕上げ

## 実施内容

- `useExecution`（`app/features/execution/useExecution.ts`）に `streamEnabled`（既定 true）と `streamRefreshDebounceMs`（既定 `DEFAULT_STREAM_REFRESH_DEBOUNCE_MS` = 500）を追加。
- SSE オン時: ストリームイベント適用後にデバウンス付きで `refreshExecutionSnapshot`（workflow + graph の GET）を実行し、Read Model を正本に寄せる。アンマウント時は再接続タイマーとデバウンス用タイマーを両方クリア。
- SSE オフ時: `EventSource` を開かず、終端状態でない間は即時 1 回の取得のあと `POLL_INTERVAL_MS`（2.5s）間隔でポーリング。
- `ExecutionHeader` に「リアルタイム更新（SSE）」チェックボックス（`streamEnabled` / `onStreamEnabledChange`）を追加。
- `ExecutionDashboard` で上記を `useExecution` に渡し、選択を `sessionStorage` キー `statevia.execution.streamEnabled`（`"1"` / `"0"`）に永続化。初回マウント後にストレージから復元。
- `useExecution` テスト: SSE オフで EventSource 非生成、ポーリング、デバウンス後 GET、および `afterEach` で `vi.useRealTimers()` を保証。
